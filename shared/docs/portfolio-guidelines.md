# Portfolio Guidelines

Use this repository as a portfolio, not just a dump of notebooks.

## Good project signals

- Clear problem statement
- Reproducible training and inference steps
- Comparison of multiple approaches
- Honest analysis of errors and trade-offs
- Short explanation of which course skills were applied

## Recommended proof for each task

- One polished `README.md`
- At least one reproducible approach
- One chart or table with metrics
- A short lessons-learned section
- Optional demo or app if the task deserves it
